## 二、8 个任务节点的 Shell 命令

### 任务 1：datax_full_sync（DataX 全量同步，hadoop101）

```
节点类型：SHELL
运行节点：hadoop101（通过 SSH）
超时：7200s
```

```bash
#!/bin/bash
dt=$(date +%Y-%m-%d)

# 1. 创建当天分区目录
for t in activity_info activity_rule base_trademark base_category1 base_category2 \
         base_category3 coupon_info sku_attr_value sku_sale_attr_value base_dic \
         sku_info base_province spu_info base_region promotion_pos promotion_refer; do
    hdfs dfs -mkdir -p /warehouse/gmall/ods/ods_${t}_full/dt=${dt}
done

# 2. 串行执行 16 张表同步
for job in /opt/module/datax/job/import/gmall.*.json; do
    tbl=$(basename $job .json | sed 's/gmall\.//')
    echo ">>> DataX: ${tbl}"
    python /opt/module/datax/bin/datax.py \
        -p"-Dtargetdir=/warehouse/gmall/ods/ods_${tbl}_full/dt=${dt}" \
        $job
    if [ $? -ne 0 ]; then
        echo "!!! ${tbl} 同步失败"
        exit 1
    fi
done

# 3. 批量注册分区
for t in activity_info activity_rule base_trademark base_category1 base_category2 \
         base_category3 coupon_info sku_attr_value sku_sale_attr_value base_dic \
         sku_info base_province spu_info base_region promotion_pos promotion_refer; do
    hive -e "MSCK REPAIR TABLE ods_${t}_full"
done
```

> DS 运行在 hadoop102，远程执行用 ssh：在 DolphinScheduler 中配置 hadoop101 为 Worker 节点，或直接在命令前加 `ssh hadoop101 "..."`

### 任务 2：dim_daily（DIM 层每日装载，hadoop100）

```
节点类型：SHELL
依赖：datax_full_sync
超时：1800s
```

```bash
#!/bin/bash
dt=$(date +%Y-%m-%d)

# dim_user_zip 读 Iceberg，需切 MR 引擎
hive -e "set hive.execution.engine=mr; INSERT OVERWRITE TABLE dim_user_zip ..."

# 7 张全量维度表 + 日期维度（Spark 引擎）
hive --hivevar dt=${dt} -f /opt/software/dim_load_daily.sql
```

> 注意：dim_user_zip 的每日装载已在 dim_load_daily.sql 中包含，上面的 MR 引擎设置仅为示例。实际执行时用 `hive --hivevar dt=${dt} -f /opt/software/dim_load_daily.sql` 即可，文件中已处理引擎切换。

### 任务 6：ads_export（ADS→MySQL 导出，hadoop101）

```
节点类型：SHELL
依赖：ads_daily
超时：1800s
```

```bash
#!/bin/bash
dt=$(date +%Y-%m-%d)

# 1. 生成 DataX 导出配置
cd /opt/module/gen_datax_config
java -jar datax-config-generator-1.0-SNAPSHOT-jar-with-dependencies.jar

# 2. 串行执行导出
for job in /opt/module/datax/job/export/gmall_report.*.json; do
    tbl=$(basename $job .json | sed 's/gmall_report\.//')
    echo ">>> 导出: ${tbl}"
    python /opt/module/datax/bin/datax.py \
        -p"-Dexportdir=/warehouse/gmall/ads/${tbl}" \
        $job
    if [ $? -ne 0 ]; then
        echo "!!! ${tbl} 导出失败"
    fi
done
```

### 任务 7：iceberg_compact（Iceberg 快照与小文件清理，hadoop100）

```
节点类型：SHELL
依赖：ads_daily
超时：1800s
```

```bash
#!/bin/bash
dt=$(date +%Y-%m-%d)
seven_days_ago=$(date -d "7 days ago" +%Y-%m-%d)

for t in cart_info comment_info coupon_use favor_info order_detail \
         order_detail_activity order_detail_coupon order_info \
         order_refund_info order_status_log payment_info \
         refund_payment user_info; do
    echo ">>> compact: ods_${t}_inc"

    hive -e "
    CALL system.rewrite_data_files(
        table => 'ods_${t}_inc',
        strategy => 'binpack',
        options => map('target-file-size-bytes', '134217728')
    );
    CALL system.expire_snapshots(
        table => 'ods_${t}_inc',
        older_than => TIMESTAMP '${seven_days_ago} 00:00:00',
        retain_last => 7
    );
    "
done
```

## 三、DolphinScheduler 中的创建步骤

### 3.1 创建项目

```
项目管理 → 创建项目 → 名称：电商离线数仓 → 描述：每日凌晨全链路调度
```

### 3.2 创建工作流

```
项目 → 工作流定义 → 创建工作流
拖拽 SHELL 节点到画布，依次配置 8 个任务节点
```

### 3.3 配置每个 SHELL 节点

| 节点 | 节点名称 | 脚本（见上） | 失败策略 |
|------|---------|-------------|:---:|
| 1 | datax_full_sync | DataX 全量同步脚本 | 结束 |
| 2 | dim_daily | dim_load_daily.sql | 结束 |
| 3 | dwd_daily | dwd_load_daily.sql | 结束 |
| 4 | dws_daily | dws_load_daily.sql | 结束 |
| 5 | ads_daily | ads_load_daily.sql | 结束 |
| 6 | ads_export | ADS→MySQL 导出 | 继续 |
| 7 | iceberg_compact | Iceberg Compaction | 继续 |

> 失败策略：前 5 个任务"结束"（核心链路失败不应继续），后 2 个"继续"（导出和 Compaction 失败不影响主链路）

### 3.4 连线（依赖关系）

```
datax_full_sync → dim_daily → dwd_daily → dws_daily → ads_daily
                                                           ├── ads_export
                                                           └── iceberg_compact
```

### 3.5 定时调度

```
上线工作流 → 定时管理 → 创建定时
cron：0 0 2 * * ? *
（每天凌晨 2 点触发）
```

## 四、注意事项

1. **Memory 资源**：任务串行执行，每个 Hive 任务仅占用一个 Spark executor (768MB)，3 台节点内存足够
2. **失败重试**：核心 5 个任务可设置重试 1-2 次，避免偶发内存不足
3. **告警**：任务 1-5 失败建议配置邮件告警
4. **Iceberg 源表引擎**：dim_daily.sql 中 dim_user_zip 已配置 `set hive.execution.engine=mr`，其余用 Spark
5. **首日数据**：首次运行前需先手动执行各层 `_load_first.sql`，日常调度只用 `_load_daily.sql`
