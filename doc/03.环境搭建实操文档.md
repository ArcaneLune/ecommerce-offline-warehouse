#### （8）编写集群分发脚本 xsync

```bash
su - hadoop
mkdir /home/hadoop/bin
cd /home/hadoop/bin
sudo vim xsync
```

脚本内容：

```bash
#!/bin/bash
# 1. 定义集群主机列表（在这里统一修改）
HOSTS="hadoop100 hadoop101 hadoop102"
# 2. 判断参数个数
if [ $# -lt 1 ]; then
  echo "Usage: $0 <file1/dir1> [file2/dir2] ..."
  exit 1
fi
# 3. 遍历集群所有机器
for host in $HOSTS; do
  echo "==================== $host ===================="
  
  # 4. 遍历所有要发送的文件/目录
  for file in "$@"; do
    # 5. 判断文件是否存在
    if [ -e "$file" ]; then
      # 6. 获取父目录（物理路径）和文件名
      pdir=$(cd -P "$(dirname "$file")" && pwd)
      fname=$(basename "$file")
      
      # 7. 远程创建目录并同步 (合并为一条 rsync 命令)
      ssh $host "mkdir -p $pdir"
      if [ $? -eq 0 ]; then
        rsync -av --rsync-path="sudo rsync" "$pdir/$fname" "$host:$pdir"
      else
        echo "ERROR: Failed to create directory on $host"
      fi
    else
      echo "ERROR: $file does not exist!"
    fi
  done
done
```

保存后添加执行权限并分发：

```bash
sudo chmod 777 /home/hadoop/bin/xsync
sudo scp -r /home/hadoop/bin hadoop101:/home/hadoop/
sudo scp -r /home/hadoop/bin hadoop102:/home/hadoop/
```

> 使用方式：`xsync /完整路径`

### 1.2 JDK 1.8 安装与配置

**部署节点**：hadoop100 / hadoop101 / hadoop102
**版本**：OpenJDK 8u492（Adoptium），[下载地址](https://mirrors.tuna.tsinghua.edu.cn/Adoptium/8/jdk/x64/linux/)

> ⚠️ **全链路硬约束**：必须使用 JDK 1.8，不可使用 JDK 11+。
>
> Flink 1.16+、Spark 3.4+、Flink CDC 3.0+ 均不兼容 JDK 8，强行运行会报 `UnsupportedClassVersionError`。

#### 操作步骤

##### 1. 卸载旧版本 JDK

```bash
rpm -qa | grep -i jdk | xargs -r rpm -e --nodeps
```

##### 2. 创建目录结构

```bash
cd /opt
sudo mkdir /opt/software   # 存放安装包
sudo mkdir /opt/module     # 软件安装目录
sudo chown -R $USER:$USER /opt
```

##### 3. 上传并解压

使用 xftp 等工具将 `OpenJDK8U-jdk_x64_linux_hotspot_8u492b09.tar.gz` 上传到 `/opt/software/`，然后：

```bash
tar -zxvf /opt/software/OpenJDK8U-jdk_x64_linux_hotspot_8u492b09.tar.gz -C /opt/module/
cd /opt/module
mv jdk8u492-b09/ jdk-1.8
```

##### 4. 配置环境变量

```bash
sudo vim /etc/profile.d/my_env.sh
```

添加以下内容：

```bash
# JAVA_HOME
export JAVA_HOME=/opt/module/jdk-1.8
export PATH=$JAVA_HOME/bin:$PATH
export CLASSPATH=.:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib
```

刷新环境变量：

```bash
source /etc/profile
```

##### 5. 分发到集群

```bash
xsync /opt/module/jdk-1.8
xsync /etc/profile.d/my_env.sh
```

> 分发后，在 hadoop101 和 hadoop102 上分别执行 `source /etc/profile`

#### 验证

```bash
java -version
# 期望输出：
# openjdk version "1.8.0_492"
# OpenJDK Runtime Environment (Temurin)(build 1.8.0_492-b09)
# OpenJDK 64-Bit Server VM (Temurin)(build 25.492-b09, mixed mode)

echo $JAVA_HOME
# 期望输出：/opt/module/jdk-1.8

javac -version
# 期望输出：javac 1.8.0_492
```

**部署节点**：hadoop100 / hadoop101 / hadoop102

| 节点 | myid |

```bash
# ZOOKEEPER_HOME
export ZOOKEEPER_HOME=/opt/module/zookeeper-3.7.1
```

刷新：

```bash
source /etc/profile
```

##### 3. 创建数据目录与 myid 文件

```bash
sudo mkdir /opt/module/zookeeper-3.7.1/zkData
sudo chown -R hadoop:hadoop /opt/module/zookeeper-3.7.1/
sudo vim /opt/module/zookeeper-3.7.1/zkData/myid
```

添加 `2`（hadoop100 的编号）。

##### 4. 配置 zoo.cfg

```bash
cd /opt/module/zookeeper-3.7.1
mv conf/zoo_sample.cfg conf/zoo.cfg
vim conf/zoo.cfg
```

修改 `dataDir`：

```
dataDir=/opt/module/zookeeper-3.7.1/zkData
```

末尾添加集群配置：

```
#######################cluster##########################
server.2=hadoop100:2888:3888
server.3=hadoop101:2888:3888
server.4=hadoop102:2888:3888
```

> 参数说明：`server.A=B:C:D`
> - A = myid 编号，与 zkData/myid 文件内容一致
> - B = 服务器地址
> - C = Follower 与 Leader 通信端口
> - D = Leader 选举端口

##### 5. JVM 内存限制

> ⚠️ 4GB 内存受限环境，ZooKeeper 的 JVM 堆必须限制在 256MB。

```bash
vim /opt/module/zookeeper-3.7.1/conf/java.env
```

```
export JVMFLAGS="-Xmx256M -Xms128M"
export ZOOKEEPER_HOME=/opt/module/zookeeper-3.7.1
```

##### 6. 分发到集群

```bash
xsync /opt/module/zookeeper-3.7.1
xsync /etc/profile.d/my_env.sh
```

> 分发后，在 hadoop101 和 hadoop102 上分别执行 `source /etc/profile`

##### 7. 修改其他节点的 myid

```bash
# hadoop101 上
sudo vim /opt/module/zookeeper-3.7.1/zkData/myid
# 改为 3

# hadoop102 上
sudo vim /opt/module/zookeeper-3.7.1/zkData/myid
# 改为 4
```

##### 8. 启动集群

```bash
# 三台分别执行
bin/zkServer.sh start
```

##### 9. 配置集群管理脚本 zk.sh

```bash
sudo vim /home/hadoop/bin/zk.sh
```

脚本内容：

```bash
#!/bin/bash
ZK_HOME="/opt/module/zookeeper-3.7.1"
ZK_BIN="$ZK_HOME/bin/zkServer.sh"
HOSTS="hadoop100 hadoop101 hadoop102"
if [ $# -ne 1 ]; then
  echo "Usage: $0 {start|stop|status}"
  exit 1
fi
case $1 in
"start")
  for i in $HOSTS; do
    echo "---------- zookeeper $i 启动 ------------"
    ssh $i "$ZK_BIN start"
  done
  ;;
"stop")
  for i in $HOSTS; do
    echo "---------- zookeeper $i 停止 ------------"
    ssh $i "$ZK_BIN stop"
  done
  ;;
"status")
  for i in $HOSTS; do
    echo "---------- zookeeper $i 状态 ------------"
    ssh $i "$ZK_BIN status"
  done
  ;;
*)
  echo "Invalid Args! Usage: $0 {start|stop|status}"
  exit 1
  ;;
esac
```

```bash
sudo chmod 777 /home/hadoop/bin/zk.sh
xsync /home/hadoop/bin/
```

##### 3. 修改配置文件

**（1）core-site.xml**

```xml
<configuration>
    <!-- 指定 NameNode 的地址 -->
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://hadoop100:8020</value>
    </property>
    <!-- 指定 hadoop 数据的存储目录 -->
    <property>
        <name>hadoop.tmp.dir</name>
        <value>/opt/module/hadoop-3.3.6/data</value>
    </property>
    <!-- 配置 HDFS 网页登录使用的静态用户为 hadoop -->
    <property>
        <name>hadoop.http.staticuser.user</name>
        <value>hadoop</value>
    </property>
    <!-- 配置该 hadoop(superUser)允许通过代理访问的主机节点 -->
    <property>
        <name>hadoop.proxyuser.hadoop.hosts</name>
        <value>*</value>
    </property>
    <!-- 配置该 hadoop(superUser)允许通过代理用户所属组 -->
    <property>
        <name>hadoop.proxyuser.hadoop.groups</name>
        <value>*</value>
    </property>
    <!-- 配置该 hadoop(superUser)允许通过代理的用户 -->
    <property>
        <name>hadoop.proxyuser.hadoop.users</name>
        <value>*</value>
    </property>
</configuration>
```

**（2）hdfs-site.xml**

```xml
<configuration>
    <!-- nn web 端访问地址 -->
    <property>
        <name>dfs.namenode.http-address</name>
        <value>hadoop100:9870</value>
    </property>
    <!-- 2nn web 端访问地址 -->
    <property>
        <name>dfs.namenode.secondary.http-address</name>
        <value>hadoop102:9868</value>
    </property>
    <!-- 测试环境指定 HDFS 副本的数量 1 -->
    <property>
        <name>dfs.replication</name>
        <value>1</value>
    </property>
</configuration>
```

> ⚠️ `dfs.replication` 设为 1（学习环境节省磁盘）。生产环境应设为 3。
>
> 📝 **备注**：`dfs.permissions.enabled` 未显式设为 false。HDFS 默认会校验文件权限，如果后续 DataX/DolphinScheduler 等组件以不同用户运行时遇到 `Permission denied`，可追加以下配置：
> ```xml
> <property>
>     <name>dfs.permissions.enabled</name>
>     <value>false</value>
> </property>
> ```

**（3）yarn-site.xml**

> ResourceManager 部署在 hadoop101，与 NameNode 分离以减轻单点压力。

```xml
<configuration>
    <!-- 指定 MR 走 shuffle -->
    <property>
        <name>yarn.nodemanager.aux-services</name>
        <value>mapreduce_shuffle</value>
    </property>
    <!-- 指定 ResourceManager 的地址 -->
    <property>
        <name>yarn.resourcemanager.hostname</name>
        <value>hadoop101</value>
    </property>
    <!-- 环境变量的继承 -->
    <property>
        <name>yarn.nodemanager.env-whitelist</name>
        <value>JAVA_HOME,HADOOP_COMMON_HOME,HADOOP_HDFS_HOME,HADOOP_CONF_DIR,CLASSPATH_PREPEND_DISTCACHE,HADOOP_YARN_HOME,HADOOP_MAPRED_HOME</value>
    </property>
    <!-- yarn 单个容器允许分配的最大最小内存（4GB 节点，留 ~2GB 给 OS 和其他进程） -->
    <property>
        <name>yarn.scheduler.minimum-allocation-mb</name>
        <value>512</value>
    </property>
    <property>
        <name>yarn.scheduler.maximum-allocation-mb</name>
        <value>2048</value>
    </property>
    <!-- yarn 容器允许管理的物理内存大小（预留 ~2GB 给 OS/ZK/Kafka/MySQL 等） -->
    <property>
        <name>yarn.nodemanager.resource.memory-mb</name>
        <value>2048</value>
    </property>
    <!-- yarn 容器允许管理的 CPU 核心数（4GB VM 通常 2-4 核，设为 2 避免超额调度） -->
    <property>
        <name>yarn.nodemanager.resource.cpu-vcores</name>
        <value>2</value>
    </property>
    <!-- 关闭 yarn 对物理内存的限制检查（4GB 节点必须关闭，避免容器被误杀） -->
```

**（7）capacity-scheduler.xml — 调大 AM 资源比例**

> ⚠️ 必须修改。YARN 容量调度器默认限制 ApplicationMaster 最多使用队列总资源的 **10%**。你的 Flink JM（1GB）作为 AM 提交到 YARN 时，单节点 2GB 队列 × 10% = 200MB，远不够 1GB → Flink Session 起不来。

```bash
vim /opt/module/hadoop-3.3.6/etc/hadoop/capacity-scheduler.xml
```

在 `<configuration>` 内添加：

```xml
<property>
    <name>yarn.scheduler.capacity.maximum-am-resource-percent</name>
    <value>0.5</value>
    <description>AM 资源占比上限。学习集群资源少，默认 0.1 不够 Flink JM(1GB) 申请。0.5 = 集群总资源的 50%，约 3GB，够 Flink JM + Spark Driver 同时存在</description>
</property>
```

> 参数说明：设为 `0.5`（50%），集群 3 节点各 2GB = 总 6GB，AM 上限 3GB。Flink JM 1GB + Spark Driver 512MB 可同时运行，Task 也还有一半资源可用。生产环境保持默认 0.1 即可。

**版本**：7.17.24，[下载地址](https://artifacts.elastic.co/downloads/beats/filebeat/filebeat-7.17.24-linux-x86_64.tar.gz)

**作用**：实时 tail 用户行为日志文件，JSON 格式校验后发送到 Kafka topic_log。Go 语言实现，不依赖 JDK，内存占用约 30MB。

#### 操作步骤

##### 1. 下载并解压

```bash
# hadoop101 上执行
cd /opt/software/
wget https://artifacts.elastic.co/downloads/beats/filebeat/filebeat-7.17.24-linux-x86_64.tar.gz
tar -zxvf filebeat-7.17.24-linux-x86_64.tar.gz -C /opt/module/
cd /opt/module
mv filebeat-7.17.24-linux-x86_64/ filebeat-7.17.24/
```

##### 2. 配置 filebeat.yml

```bash
cd /opt/module/filebeat-7.17.24/
vim filebeat.yml
```

```yaml
# ===== 输入：tail 日志文件，JSON 解析 =====
filebeat.inputs:
- type: log
  enabled: true
  paths:
    - /data/logs/app/*.log
  json.keys_under_root: true
  json.add_error_key: true
  json.overwrite_keys: true
  tags: ["user_behavior"]

# ===== 输出：Kafka topic_log =====
output.kafka:
  hosts: ["hadoop100:9092", "hadoop101:9092", "hadoop102:9092"]
  topic: "topic_log"
  partition.round_robin:
    reachable_only: false
  required_acks: 1
  compression: gzip
  max_message_bytes: 1000000
```

> 参数说明：
> - `json.keys_under_root: true` — JSON 字段直接提升为顶层字段，方便 Kafka 消费端解析
> - `json.add_error_key: true` — JSON 解析失败的条目会被标记，可在 Kafka 消费端过滤排查
> - `topic: "topic_log"` — 对应 Kafka 已创建的 topic_log
> - 日志路径 `/data/logs/app/*.log` 为后续业务数据目录，当前阶段目录不存在不影响 Filebeat 启动

##### 3. 验证 + 配置 systemd 服务

```bash
# 前台启动测试（确认能连上 Kafka，Ctrl+C 退出）
cd /opt/module/filebeat-7.17.24/
./filebeat -e

# 创建 systemd 服务（替代 nohup，开机自启 + 异常重启）
sudo tee /etc/systemd/system/filebeat.service << 'EOF'
[Unit]
Description=Filebeat - User Behavior Log Collector
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=hadoop
Group=hadoop
WorkingDirectory=/opt/module/filebeat-7.17.24
ExecStartPre=/opt/module/filebeat-7.17.24/filebeat test config
ExecStart=/opt/module/filebeat-7.17.24/filebeat -e
Restart=on-failure
RestartSec=10
TimeoutStopSec=10
MemoryMax=256M

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl start filebeat
sudo systemctl status filebeat
```

> Filebeat 日志位置：`/opt/module/filebeat-7.17.24/logs/filebeat`（滚动日志文件）

#### 操作步骤

##### 1. 上传并解压

将 `flink-1.15.4-bin-scala_2.12.tgz` 上传到 hadoop100 的 `/opt/software/`。

```bash
tar -zxvf /opt/software/flink-1.15.4-bin-scala_2.12.tgz -C /opt/module/
```

##### 2. 配置环境变量

```bash
sudo vim /etc/profile.d/my_env.sh
```

末尾添加：

```bash
# FLINK_HOME
export FLINK_HOME=/opt/module/flink-1.15.4
export PATH=$PATH:$FLINK_HOME/bin
# YARN 集成（必须，否则 Flink 找不到 Hadoop）
```

> ⚠️ `HADOOP_CLASSPATH` 必须在 OS 环境变量中设置，`flink-conf.yaml` 中的 `env.hadoop.conf.dir` **不会生效**。
>
> 验证 Classpath：
> ```bash
> echo $HADOOP_CLASSPATH | tr ':' '\n' | head -5
> ```

##### 3. 配置 flink-conf.yaml（YARN + 内存调优）

```bash
vim /opt/module/flink-1.15.4/conf/flink-conf.yaml
```

修改以下参数：

```yaml
# ===== 基础配置 =====
jobmanager.rpc.address: localhost
jobmanager.rpc.port: 6123
jobmanager.bind-host: 0.0.0.0
rest.address: 0.0.0.0
rest.bind-address: 0.0.0.0
# ===== 内存（4GB 节点约束） =====
jobmanager.memory.process.size: 1024m
taskmanager.memory.process.size: 1024m
taskmanager.numberOfTaskSlots: 2
# ===== 并行度 =====
parallelism.default: 2
# ===== 容错 =====
jobmanager.execution.failover-strategy: region
# ===== Checkpoint（CDC 必须） =====
execution.checkpointing.interval: 120000
execution.checkpointing.mode: EXACTLY_ONCE
execution.checkpointing.timeout: 600000
execution.checkpointing.min-pause: 30000
execution.checkpointing.max-concurrent-checkpoints: 1
state.backend: rocksdb
state.backend.incremental: true
state.checkpoints.dir: hdfs://hadoop100:8020/flink/checkpoints
state.savepoints.dir: hdfs://hadoop100:8020/flink/savepoints
state.backend.rocksdb.memory.managed: true
state.backend.rocksdb.memory.write-buffer-ratio: 0.3
state.backend.rocksdb.memory.high-prio-pool-ratio: 0.2
# ===== History Server（Job 结束后仍可回看执行记录） =====
jobmanager.archive.fs.dir: hdfs://hadoop100:8020/flink-history
historyserver.web.address: 0.0.0.0
historyserver.web.port: 8082
historyserver.archive.fs.dir: hdfs://hadoop100:8020/flink-history
historyserver.archive.fs.refresh-interval: 10000
# ===== YARN 资源配置 =====
yarn.application-attempts: 3
yarn.classpath.include-user-jar: FIRST
```

> 参数说明：
> - `bind-host/rest.address: 0.0.0.0` — YARN 容器 IP 随机分配，必须绑定所有接口
> - JM 1024m + TM 1024m × 1 ≈ 2GB，4GB 节点可容纳
> - `taskmanager.numberOfTaskSlots: 2` — CDC→Kafka 和 Kafka→Iceberg 各占 1 个 slot
> - RocksDB 增量 checkpoint，适合 4GB 节点
> - History Server 端口 8082，避免与 YARN RM（8088）冲突
> - **未配置 HA**：HDFS 是单 NameNode，Flink HA 无实际意义

##### 4. 创建 HDFS 目录

```bash
hdfs dfs -mkdir -p /flink/checkpoints /flink/savepoints /flink-history
```

##### 5. 启动 YARN Session + History Server

```bash
# 启动 YARN Session（1 个 TM，含 2 个 slot 足够两个 CDC Job 共用）
yarn-session.sh -d -nm FlinkSession -s 1 -tm 1024
# 记下输出的 application ID，如 application_xxx_0001

# 启动 History Server
/opt/module/flink-1.15.4/bin/historyserver.sh start
```

> `-s 1` 只起 1 个 TaskManager，4GB 节点只能容纳 1 个 TM 容器。TM 内 2 个 slot，两个 CDC Job 各占 1 个。

##### 6. 查看与停止

```bash
# 查看 Session 中运行的任务
flink list -m yarn-cluster -yid <application_id>
# Flink Web UI：http://hadoop101:8088 → FlinkSession → ApplicationMaster
# History Server：http://hadoop100:8082
# 停止
yarn application -kill <application_id>
```

> ⚠️ 提交和查看命令必须使用 `-m yarn-cluster -yid`，不可用 `-t yarn-session`。Flink 1.15 的 `-t yarn-session` 模式下 `-yid` 参数不生效，clusterID 解析为 null。

#### 验证

```bash

# 2. 提交 WordCount 测试

# 输出类似：Job has been submitted with JobID xxxx
# 等待约 1 分钟后显示：Job with JobID xxxx has finished.
# Job Runtime: ~90000 ms
```

> ⚠️ **DataX 跨节点注意**：DataX 实际安装在 hadoop101，而 DS Worker 在 hadoop102 上执行任务，它会在本地（hadoop102）查找 `/opt/module/datax`。调度 DataX 任务时有三种方案：
> 1. 在 hadoop102 上也安装一份 DataX（解压即用，不占内存）
> 2. DS 任务配置为 SSH 远程执行（`ssh hadoop101 "python /opt/module/datax/bin/datax.py ..."`）
> 3. 后续创建 HDFS 全量同步任务时选择方案 2，最为简单。

##### 7. 限制 JVM 堆内存

> ⚠️ 4GB 节点必须限制，否则 OS 会杀进程。DS 每个服务默认堆 1GB-4GB，4 个服务叠加远超 4GB。

通过 `jvm_args_env.sh` 限制：

```bash
# Master (384MB heap + 限制非堆)
sudo tee /opt/module/apache-dolphinscheduler-3.1.9-bin/master-server/bin/jvm_args_env.sh << 'EOF'
-Xms256m
-Xmx384m
-XX:MaxMetaspaceSize=128m
-XX:ReservedCodeCacheSize=64m
-Xss256k
EOF
# Worker
sudo tee /opt/module/apache-dolphinscheduler-3.1.9-bin/worker-server/bin/jvm_args_env.sh << 'EOF'
-Xms256m
-Xmx384m
-XX:MaxMetaspaceSize=128m
-XX:ReservedCodeCacheSize=64m
-Xss256k
EOF
# API
sudo tee /opt/module/apache-dolphinscheduler-3.1.9-bin/api-server/bin/jvm_args_env.sh << 'EOF'
-Xms192m
-Xmx256m
-XX:MaxMetaspaceSize=128m
-XX:ReservedCodeCacheSize=64m
-Xss256k
EOF
# Alert
sudo tee /opt/module/apache-dolphinscheduler-3.1.9-bin/alert-server/bin/jvm_args_env.sh << 'EOF'
-Xms128m
-Xmx128m
EOF
```

> 四项 JVM 合计约 1.4GB，加上 ZK+DN+NM+Kafka+SNN+Superset ≈ 3.4GB，4GB 节点勉强存活。

```bash
#!/bin/bash
case $1 in
"start")
    source /opt/module/superset-env/bin/activate
    if ps -ef | grep -v grep | grep -q "gunicorn.*superset"; then
        echo "Superset 已在运行"
    else
        gunicorn --workers 3 --timeout 120 --bind 0.0.0.0:8088 \
          --daemon "superset.app:create_app()"
        echo "Superset 已启动"
    fi
;;
"stop")
    ps -ef | grep "gunicorn.*superset" | grep -v grep | awk '{print $2}' | xargs kill -9 2>/dev/null
    echo "Superset 已停止"
;;
"status")
    if ps -ef | grep -v grep | grep -q "gunicorn.*superset"; then
        echo "Superset 运行中"
    else
        echo "Superset 未运行"
    fi
;;
*)
    echo "用法: superset.sh {start|stop|status}"
;;
esac
```
