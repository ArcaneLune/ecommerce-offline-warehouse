# 电商离线数仓 —— 业务数据增量同步实操文档

> 记录业务数据从 MySQL binlog 到 Iceberg ODS 层（13 张增量表）的完整制作过程、验证方法以及遇到的坑和报错。
> 最后更新：2026-06-25

---

## 一、增量同步链路总览

### 1.1 数据流向

```
首日（一次性）：
MySQL 13 张表 → Flink CDC (initial 全量快照) → Kafka topic_db → Flink Job 2 → Iceberg INSERT

日常（持续）：
MySQL binlog → Flink Job 1 (CDC) → Kafka topic_db → Flink Job 2 → Iceberg INSERT
```

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    业务数据 —— 增量同步 (持续)                            │
│                                                                         │
│  MySQL binlog → Flink Job1(CDC) → Kafka(topic_db)                      │
│                                   → Flink Job2 → Iceberg INSERT         │
│  两个 Job 共享 Flink 1.15.4 Session 集群, 均运行在 YARN                  │
│  Iceberg 表: /warehouse/gmall/ods/default/{表名}_inc/                   │
│  13 张, v2 格式 + Parquet                                              │
│  Hive 管 Iceberg DDL, Flink 只做 INSERT（共享 Hive Metastore）          │
└─────────────────────────────────────────────────────────────────────────┘
```

### 1.2 关键组件

| 组件 | 版本 | 节点 | 角色 |
|------|------|:---:|------|
| MySQL | 8.0.39 | hadoop101 | 业务数据源，开启 binlog (ROW) |
| Flink | 1.15.4 | YARN 集群 | 计算引擎 |
| Flink CDC | 2.3.0 | $FLINK_HOME/lib/ | MySQL binlog 实时采集 |
| Kafka | 3.6.2 | 三节点 | topic_db 消息队列中转 |
| Iceberg | 1.3.1 | HDFS | ODS 增量表存储（v2 + Parquet） |
| Hive | 3.1.3 | hadoop100 | Iceberg 表 DDL（STORED BY）+ 查询 |

### 1.3 核心架构：Hive 管 DDL，Flink 只写数据

**Hive CLI**：CREATE TABLE / DROP TABLE（`STORED BY 'org.apache.iceberg.mr.hive.HiveIcebergStorageHandler'`）
**Flink**：INSERT INTO（通过 HiveCatalog 对已存在表做 INSERT，永不建表）

```
            Hive Metastore (thrift://hadoop100:9083)
           ┌──────────────────────────────────────┐
           │  ods_user_info_inc 元数据              │
           └──────┬───────────────┬───────────────┘
                  │               │
        ┌─────────▼────┐   ┌─────▼──────────┐
        │  Hive 3.1.3   │   │  Flink 1.15.4  │
        │  CREATE TABLE │   │  INSERT INTO   │
        │  STORED BY    │   │  (HiveCatalog) │
        └───────────────┘   └────────────────┘
                  │               │
                  └───────┬───────┘
              ┌───────────▼───────────┐
              │  HDFS Iceberg 数据文件  │
              │  /warehouse/gmall/ods/ │
              │  default/ods_xxx_inc/  │
              └───────────────────────┘
```

| 维度 | 全量同步 | 增量同步 |

| 运行模式 | 每日定时 | YARN 常驻 |

## 四、第三步：Flink 环境配置与 Session 启动

** 执行节点**：hadoop100

### flink-conf.yaml 必须配置

```bash
cat >> $FLINK_HOME/conf/flink-conf.yaml << 'EOF'
taskmanager.memory.managed.size: 64m
taskmanager.memory.jvm-metaspace.size: 128m
classloader.resolve-order: parent-first
execution.checkpointing.unaligned.enabled: false
EOF
```

| 参数 | 值 | 原因 |
|------|------|------|
| `managed.size` | 64m | 流作业不用 RocksDB |
| `jvm-metaspace.size` | 128m | 防止元空间挤占堆 |
| `classloader.resolve-order` | parent-first | 防止 JAR 冲突 |
| `checkpointing.unaligned` | false | Iceberg 要求 aligned checkpoint |

### 启动 Session

```bash
yarn application -list | grep FlinkSession && yarn application -kill $(yarn application -list | grep FlinkSession | awk '{print $1}')
rm -rf /tmp/flink-*
sleep 2

yarn-session.sh -d -nm FlinkSession -s 2 -tm 2048
sleep 5
yarn application -list | grep FlinkSession
```

| 参数 | 值 | 原因 |
|------|------|------|
| `-s 2` | 2 个 slot | 同时跑 CDC + Iceberg |
| `-tm 2048` | TM 2GB | 1GB 不够，ParquetWriter 需要 100MB+ |

### Flink Hive Metastore JAR 依赖

```bash
ls $FLINK_HOME/lib/ | grep -E "hive-metastore|libfb303|hive-common|hive-serde|hive-exec|flink-connector-hive"

# 如果缺少：
cp $HIVE_HOME/lib/hive-metastore-3.1.3.jar $FLINK_HOME/lib/
cp $HIVE_HOME/lib/libfb303-0.9.3.jar $FLINK_HOME/lib/
cp $HIVE_HOME/lib/hive-common-3.1.3.jar $FLINK_HOME/lib/
cp $HIVE_HOME/lib/hive-serde-3.1.3.jar $FLINK_HOME/lib/
cp $HIVE_HOME/lib/hive-exec-3.1.3.jar $FLINK_HOME/lib/
```

## 五、第四步：Iceberg 增量表建表（13 张）

** 执行节点**：hadoop100
** 工具**：`hive` CLI

### 操作步骤

```bash
# 1. 创建 HDFS 目录
for t in cart_info comment_info coupon_use favor_info order_detail \
         order_detail_activity order_detail_coupon order_info \
         order_refund_info order_status_log payment_info \
         refund_payment user_info; do
    hdfs dfs -mkdir -p /warehouse/gmall/ods/default/ods_${t}_inc
done

# 2. 执行 DDL（STORED BY Iceberg StorageHandler）
hive -f /opt/software/ods_inc_iceberg_ddl.sql

# 3. 验证
hive -e "SHOW TABLES" | grep _inc | wc -l
# 期望：13
```

### DDL 格式（以 user_info 为例）

```sql
CREATE TABLE ods_user_info_inc (
  id BIGINT, login_name STRING, nick_name STRING, passwd STRING, name STRING,
  phone_num STRING, email STRING, head_img STRING, user_level STRING, birthday STRING,
  gender STRING, create_time STRING, operate_time STRING
) PARTITIONED BY (dt STRING)
STORED BY 'org.apache.iceberg.mr.hive.HiveIcebergStorageHandler'
LOCATION 'hdfs://hadoop100:8020/warehouse/gmall/ods/default/ods_user_info_inc'
TBLPROPERTIES ('write.format.default'='parquet','format-version'='2');
```

## 六、第五步：Flink 执行 SQL（CDC → Kafka → Iceberg）

** 执行节点**：hadoop100
** 文件**：`cdc_job1_paste.sql`（27 条）+ `cdc_job2_paste.sql`（15 条）

### SQL 文件结构

| 文件 | 条数 | 内容 |
|------|:---:|------|
| `cdc_job1_paste.sql` | 27 | kafka_sink(1) + CDC Source(13) + INSERT 到 Kafka(13) |
| `cdc_job2_paste.sql` | 15 | kafka_source(1) + CREATE CATALOG(1) + INSERT 到 Iceberg(13) |

### 操作步骤

```bash
app_id=$(yarn application -list | grep FlinkSession | awk '{print $1}')
sql-client.sh -m yarn-cluster -yid $app_id
```

在 SQL Client 中**逐条粘贴**执行（不能用 SOURCE 命令）：

**阶段 A**：粘贴 `cdc_job1_paste.sql` 全部 27 条
**阶段 B**：等 30s（CDC initial snapshot）→ 粘贴 `cdc_job2_paste.sql` 全部 15 条
**阶段 C**：等 30s（checkpoint）→ `QUIT`

> Flink 1.15.4 **不支持** `SHOW JOBS`。用 Kafka 消息数判断 CDC 进度：`kafka-run-class.sh kafka.tools.GetOffsetShell --topic topic_db --bootstrap-server hadoop100:9092 --time -1`

### 验证

```bash
sleep 10
hive -e "SELECT count(*) FROM ods_user_info_inc"
hive -e "SELECT * FROM ods_user_info_inc LIMIT 3"
```

### 🕳️ 问题记录（共 11 个，从原始 Spark 架构试错到最终 Hive 架构稳定）

| 序号 | 问题 | 根因 | 解决方案 |
|------|------|------|---------|
| 1 | Flink CREATE TABLE 后 Hive 查不到数据 | HIVE-26576 | Flink 永不建表 |
| 2 | HadoopCatalog 有 snapshot 无 parquet | Iceberg 1.3.1 + Flink 1.15.4 bug | 用 HiveCatalog |
| 3 | `NoSuchObjectException` | 缺 Hive Metastore JAR | 补 5 个 JAR 到 $FLINK_HOME/lib |
| 4 | `Equality field columns empty` | Flink Upsert 需要 PK | 纯 INSERT |
| 5 | `catalog-name cannot be null` | CREATE TABLE 缺参数 | HiveCatalog 直接 INSERT |
| 6 | 路径不一致 | Flink CREATE CATALOG 缺 warehouse | `'warehouse'='hdfs://hadoop100:8020/warehouse'` |
| 7 | HDFS 有 parquet 但 hive 查不到 | 手动 kill → checkpoint 未提交 | QUIT 退出；用 hive 重建表恢复 |
| 8 | `GC overhead limit exceeded` | TM 1GB heap 25MB | `-tm 2048` |
| 9 | `Java heap space`(HeapBytesCompressor) | PARQUET-2166 | `-tm 2048` |
| 10 | `RecipientUnreachableException` 反复重启 | managed 占 40% 超容器 | `managed.size: 64m` + `metaspace: 128m` |
| 11 | `StreamCorruptedException: unread block data` | classloader 冲突 | `classloader.resolve-order: parent-first` |

### 7.4 孤儿文件修复

```
现象：HDFS 有 parquet，hive SELECT count(*) = 0
诊断：hive -e "SELECT * FROM ods_xxx_inc.snapshots" → 空
根因：手动 kill Flink → checkpoint 未提交
```

```bash
# 修复：删表重建 + 重新增量同步
hive -e "DROP TABLE IF EXISTS ods_xxx_inc"
hdfs dfs -rm -r -skipTrash /warehouse/gmall/ods/default/ods_xxx_inc
hdfs dfs -mkdir -p /warehouse/gmall/ods/default/ods_xxx_inc
hive -e "CREATE TABLE ods_xxx_inc (...) STORED BY 'org.apache.iceberg.mr.hive.HiveIcebergStorageHandler' ..."
# 然后重新跑 CDC → Kafka → Iceberg
```

### 7.5 正确停止 Flink

```
QUIT → 等 30s → yarn application -kill <app_id>
```

**永远不要**直接 `Ctrl+C` 或 `yarn application -kill`，数据会丢失。

## 九、附录：Flink 内存调优速查

### 最终配置

```yaml
# flink-conf.yaml
taskmanager.memory.managed.size: 64m
taskmanager.memory.jvm-metaspace.size: 128m
classloader.resolve-order: parent-first
execution.checkpointing.unaligned.enabled: false
```

Session：`yarn-session.sh -d -nm FlinkSession -s 2 -tm 2048`

### 错误速查

| 错误关键词 | 根因 | 修复 |
|-----------|------|------|
| `unread block data` | classloader | `parent-first` |
| `GC overhead limit exceeded` | heap 小 | `-tm 2048` |
| `Java heap space` + ParquetWriter | PARQUET-2166 | `-tm 2048` |
| `RecipientUnreachableException` 反复 RESTARTING | 总内存超容器 | `managed.size: 64m` |
| HDFS 有 parquet 但查询 0 行 | checkpoint 未触发 | QUIT 退出，不 kill |
