## 二、7 个任务节点的 Shell 命令

### 任务 1：datax_full_sync（DataX 全量同步，hadoop101）

| 配置项 | 值 |
|--------|-----|
| 节点类型 | SHELL |
| 运行节点 | hadoop101 |
| 超时 | 7200s（2 小时） |
| 失败策略 | 结束 |
| 重试次数 | 1 |

```bash
#!/bin/bash
dt=$(date +%Y-%m-%d)

# 1. 创建当天分区目录
for t in activity_info activity_rule base_trademark base_category1 base_category2 \
         base_category3 coupon_info sku_attr_value sku_sale_attr_value base_dic \
         sku_info base_province spu_info base_region promotion_pos promotion_refer; do
    hdfs dfs -mkdir -p /warehouse/gmall/ods/ods_${t}_full/dt=${dt}
done

# 2. DataX JSON 由配置生成器提前生成好，调度时直接串行执行
for job in /opt/module/datax/job/import/gmall.*.json; do
    tbl=$(basename $job .json | sed 's/gmall\.//')
    echo ">>> DataX: ${tbl}"
    python /opt/module/datax/bin/datax.py \
        -p"-Dtargetdir=/warehouse/gmall/ods/ods_${tbl}_full/dt=${dt}" \
        $job
    if [ $? -ne 0 ]; then
        echo "!!! ${tbl} 同步失败，终止流程"
        exit 1
    fi
done

# 3. 批量注册分区（DataX 写入不会自动注册）
for t in activity_info activity_rule base_trademark base_category1 base_category2 \
         base_category3 coupon_info sku_attr_value sku_sale_attr_value base_dic \
         sku_info base_province spu_info base_region promotion_pos promotion_refer; do
    hive -e "MSCK REPAIR TABLE ods_${t}_full"
done
```

### 任务 6：ads_export（ADS → MySQL 导出，hadoop101）

| 配置项 | 值 |
|--------|-----|
| 节点类型 | SHELL |
| 依赖 | ads_daily |
| 超时 | 1800s（30 分钟） |
| 失败策略 | **继续** |
| 重试次数 | 0 |

```bash
#!/bin/bash

# 1. 用自建 JAR 生成最新 DataX 导出配置
cd /opt/software/datax-generator/
java -jar datax-config-generator-1.0-SNAPSHOT-jar-with-dependencies.jar

# 2. 串行执行 16 张表导出
for job in /opt/module/datax/job/export/gmall_report.*.json; do
    tbl=$(basename $job .json | sed 's/gmall_report\.//')
    echo ">>> 导出: ${tbl}"
    python /opt/module/datax/bin/datax.py \
        -p"-Dexportdir=/warehouse/gmall/ads/${tbl}" \
        $job
    if [ $? -ne 0 ]; then
        echo "!!! ${tbl} 导出失败，继续下一张"
    fi
done
```

> 失败策略设为"继续"：导出失败不影响数仓主链路，次日重跑即可覆盖。

### 任务 7：iceberg_compact（Iceberg 快照与小文件清理，hadoop100）

| 配置项 | 值 |
|--------|-----|
| 节点类型 | SHELL |
| 依赖 | ads_daily |
| 超时 | 1800s（30 分钟） |
| 失败策略 | **继续** |
| 重试次数 | 0 |

```bash
#!/bin/bash
seven_days_ago=$(date -d "7 days ago" +%Y-%m-%d)

for t in cart_info comment_info coupon_use favor_info order_detail \
         order_detail_activity order_detail_coupon order_info \
         order_refund_info order_status_log payment_info \
         refund_payment user_info; do
    echo ">>> compact: ods_${t}_inc"

    hive -e "
    set hive.execution.engine=spark;
    CALL system.rewrite_data_files(
        table => 'ods_${t}_inc',
        strategy => 'binpack',
        options => map('target-file-size-bytes', '134217728')
    );
    CALL system.expire_snapshots(
        table => 'ods_${t}_inc',
        older_than => TIMESTAMP '${seven_days_ago} 00:00:00',
        retain_last => 7
    );
    "
done
```

> Iceberg Compaction 需要在 Spark 引擎下执行（Iceberg stored procedure 通过 Spark 扩展加载）。失败策略设为"继续"：compaction 失败不阻塞主链路，次日凌晨再次尝试。

## 四、引擎切换策略

| 阶段 | 引擎 | 原因 |
|------|:---:|------|
| dim_user_zip（DIM 层） | **MapReduce** | 源表 ods_user_info_inc 是 Iceberg，Spark 向量化读取会 OOM |
| 其余 DIM/DWD/DWS/ADS | **Spark** | 源表均为 Hive ORC 原生表，Spark 引擎效率高 |
| Iceberg Compaction | **Spark** | `CALL system.xxx` 需要通过 Spark Iceberg 扩展执行 |

> 引擎切换均已内置在对应 SQL 文件中（`set hive.execution.engine=mr/spark`），DS 任务节点只需 `hive -f` 即可，无需额外处理。

## 五、Flink CDC 任务（独立管理，不在此工作流内）

Flink 的 Job1（CDC→Kafka）和 Job2（Kafka→Iceberg）是 **7×24 持续运行的长服务**，在 DolphinScheduler 之外独立管理：

```bash
# 启动 YARN Session（只需一次，重启后需重新执行）
yarn-session.sh -d -nm FlinkSession -s 2 -tm 2048

# Job1：CDC → Kafka（持续运行）
sql-client.sh embedded -f /opt/software/cdc_job1.sql

# Job2：Kafka → Iceberg（持续运行）
sql-client.sh embedded -f /opt/software/cdc_job2.sql
```

> Flink Job 不在这个每日 DAG 中。它们是常驻的流处理任务，不需要每天启停。仅 YARN Session 重启时需要手动重新提交。

## 六、故障处理

| 场景 | 处理方式 |
|------|---------|
| datax_full_sync 失败 | 流程终止，手动排查 MySQL/DataX 问题，修复后重跑 |
| dim/dwd/dws/ads 失败 | 流程终止，手动排查 Hive SQL 报错，修复后从失败节点重跑 |
| ads_export 失败 | 流程继续，次日凌晨重新导出会全量覆盖 |
| iceberg_compact 失败 | 流程继续，小文件积压不会影响数据正确性，次日再执行 |
| Flink CDC Job 中断 | 登录 hadoop100，重新提交 YARN Session + Job |

## 七、注意事项

1. **首日数据**：首次运行前需先手动执行各层 `_load_first.sql`，日常调度只用 `_load_daily.sql`
2. **内存资源**：任务串行执行，每个 Hive 任务仅占用一个 Spark executor (768MB)，不会叠加
3. **告警**：建议任务 1-5 配置邮件/钉钉告警，6-7 可关掉告警减少噪声
4. **DS Worker 节点**：DolphinScheduler 安装在 hadoop102，SHELL 任务默认在 hadoop102 本地执行。如果 DS 没有配置 hadoop100/hadoop101 作为 Worker，需在 SHELL 脚本前加 `ssh hadoop100 "..."` 或 `ssh hadoop101 "..."`
5. **Iceberg Compaction 频率**：每日一次 binpack 合并已足够，如果 Flink 写入频率很高可改为每 12 小时一次
