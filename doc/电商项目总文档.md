## 一、项目概述

### 1.1 项目定位

电商大数据离线数仓项目，模拟从数据产生到可视化的完整链路。

### 1.2 数据来源

- **用户行为日志**：模拟生成的 JSON 日志文件（页面日志 + 启动日志）
- **业务数据库**：MySQL 中的业务表（商品、订单、用户、活动等）

### 1.3 技术选型原则

- 全链路 JDK 1.8 兼容
- 资源受限环境（3 × 5GB 虚拟机）下的最优方案

## 二、环境与约束

### 2.1 硬件环境

| 项目 | 配置 |
|------|------|
| 虚拟机数量 | 3 台 |
| 操作系统 | CentOS 7.9 |
| 内存 | 每台 5GB |
| 磁盘 | 每台 50GB |
| JDK | **1.8** (硬约束) |

### 2.2 节点规划

| 节点 | 主机名 | 常驻进程 |
|------|--------|---------|
| 节点 1 | hadoop100 | NameNode, DataNode, NodeManager, ZooKeeper, Kafka, Hive Metastore, Spark(3.3.1), Flink客户端 |
| 节点 2 | hadoop101 | DataNode, ResourceManager, NodeManager, ZooKeeper, Kafka, MySQL, SeaTunnel, Filebeat, DataX |
| 节点 3 | hadoop102 | SecondaryNameNode, DataNode, NodeManager, ZooKeeper, Kafka, DolphinScheduler, Superset |

> Flink 两个 Job（CDC→Kafka, Kafka→Iceberg）运行在 YARN 上，不绑定特定节点。

### 2.3 关键约束

- **JDK 1.8** 决定所有组件的版本上限（见第五章版本锁定清单）
- **5GB 内存** 决定常驻进程必须精简——每个 JVM 进程堆内存控制在 512MB 以内
- **50GB 磁盘** HDFS 需预留至少 20GB，Kafka 日志和 MySQL 数据各约 5GB

- **集群**：三台节点均部署，通过配置文件区分角色

**节点负载概览**：

| 节点 | 常驻进程数 | 内存压力 | 说明 |
|------|:---:|:---:|------|
| hadoop100 | NN + DN + NM + ZK + Kafka + Hive Metastore + HiveServer2 | 中等 | HDFS 主节点，Spark 3.3.1 + Flink客户端 |
| hadoop101 | DN + RM + NM + ZK + Kafka + MySQL + Filebeat + SeaTunnel + DataX | **较高** | 数据中转枢纽 + 全量同步 |
| hadoop102 | SNN + DN + NM + ZK + Kafka + DolphinScheduler + Superset | 中等 | 调度与可视化 |

## 四、全链路架构

### 4.1 架构全景图

```
┌─────────────────────────────────────────────────────────────────────────┐
│                           日志数据路径                                    │
│                                                                         │
│  日志文件(.log)  →  Filebeat  →  Kafka  →  SeaTunnel  →  HDFS           │
│  (页面+启动)      (tail+校验)   (topic_log)  (STREAMING)  /warehouse/    │
│                  Go, ~30MB                JVM, ~200MB   ods/ods_log_inc/   │
│                                                         dt=YYYY-MM-DD/     │
│                                                         JSON SerDe表       │
└─────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────┐
│                    业务数据 —— 全量同步 (每日)                            │
│                                                                         │
│  MySQL  →  DataX  →  HDFS /warehouse/ods/{表名}_full/dt=YYYY-MM-DD/     │
│           (DolphinScheduler 凌晨调度)  Hive 原生表, 16 张                │
└─────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────┐
│                    业务数据 —— 增量同步 (持续)                            │
│                                                                         │
│  ┌──── 首日 ────────────────────────────────────────────────────┐       │
│  │ Flink CDC (initial 模式) → 全量快照 → Kafka → Flink Job 2     │       │
│  │ → Iceberg Upsert (13 张表初始化)                              │       │
│  └──────────────────────────────────────────────────────────────┘       │
│                                                                         │
│  ┌──── 日常 ────────────────────────────────────────────────────┐       │
│  │ MySQL binlog → Flink Job1(CDC) → Kafka(topic_db)             │       │
│  │                                  → Flink Job2 → Iceberg Upsert│       │
│  │  两个 Job 共享 Flink 1.15.4 Session 集群, 均运行在 YARN        │       │
│  │  Iceberg 表: /warehouse/ods/{表名}_inc/                       │       │
│  │  13 张, v2 格式, write.upsert.enabled=true                   │       │
│  └──────────────────────────────────────────────────────────────┘       │
└─────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────┐
│                          数仓计算层 (每日凌晨)                            │
│                                                                         │
│  ODS (18 Hive原生 + 13 Iceberg)                                         │
│      │                                                                  │
│      ▼  Spark SQL 3.3.1 on YARN                                        │
│  DIM 层                                                                 │
│      │                                                                  │
│      ▼                                                                  │
│  DWD 层 (页面日志展开: actions / displays / err)                         │
│      │                                                                  │
│      ▼                                                                  │
│  DWS 层 (轻度汇总)                                                       │
│      │                                                                  │
│      ▼                                                                  │
│  ADS 层 (应用指标)                                                       │
│      │                                                                  │
│      ▼  DataX                                                           │
│  MySQL (结果表)                                                          │
│      │                                                                  │
│      ▼                                                                  │
│  Superset (可视化)                                                       │
│                                                                         │
│  全链路调度: DolphinScheduler 3.1.9                                      │
│  元数据中心: Hive Metastore (Hive表 + Iceberg表 统一管理)                 │
└─────────────────────────────────────────────────────────────────────────┘
```

### 4.2 数据流转总结

| 路径 | 源 | 经过 | 目标 | 频率 |
|------|-----|------|------|------|
| 日志 | .log 文件 | Filebeat → Kafka → SeaTunnel | HDFS (ods_log_inc) | 持续 |
| 业务全量 | MySQL | DataX | HDFS (Hive原生, 16张) | 每日 |
| 业务增量(首日) | MySQL | Flink CDC initial → Kafka → Flink Job2 | Iceberg (13张) | 一次性 |
| 业务增量(日常) | MySQL binlog | Flink CDC → Kafka → Flink Job2 | Iceberg Upsert (13张) | 持续 |
| 数仓计算 | ODS | Spark SQL | DIM→DWD→DWS→ADS | 每日 |
| 结果导出 | ADS | DataX | MySQL | 每日 |

> 核心原则：所有组件必须在 JDK 1.8 下稳定运行。

| 组件 | 版本 | JDK 8 支持 | 角色 | 关键依赖 |
|------|------|-----------|------|---------|
| **Flink** | **1.15.4** | ✅ 官方支持 | CDC + Iceberg 运行时 | 最后支持 JDK 8 的 Flink |
| **Flink CDC** | **2.3.0** | ✅ 官方支持 | MySQL → Kafka | 支持 Flink 1.15, Java 8+ |
| **Iceberg** | **1.3.1** | ✅ 可用 | ODS 增量表格式 | Flink 端 + Spark 端共用 |

| **Kafka** | **3.6.2** | ✅ 官方支持 | 消息队列 | |

| **SeaTunnel** | **2.3.13** | ✅ 官方支持 | 日志→HDFS | |

### 5.2 JAR 包清单

```bash
# ========== Flink $FLINK_HOME/lib/ ==========
flink-sql-connector-mysql-cdc-2.3.0.jar        # CDC connector (MySQL binlog)
iceberg-flink-runtime-1.15-1.3.1.jar            # Iceberg Flink 集成
flink-sql-connector-kafka-1.15.4.jar            # Kafka connector

# ========== Spark $SPARK_HOME/jars/ ==========
iceberg-spark-runtime-3.3_2.12-1.3.1.jar        # Iceberg Spark 集成
spark-sql-kafka-0-10_2.12-3.3.1.jar             # Kafka 数据源 (如需)

# ========== SeaTunnel $SEATUNNEL_HOME/plugins/ ==========
connector-kafka-2.3.13.jar                        # Kafka source
connector-hdfs-file-2.3.13.jar                    # HDFS sink
```

### 5.3 版本选择依据

**Flink 1.15.4**：Flink 1.15 是官方声明支持 JDK 8 的最后一个大版本，1.15.4 是该线的最后一个 bugfix 版本（2024 年 3 月发布）。Flink 1.16+ 移除 JDK 8 支持，强制要求 JDK 11。

**Flink CDC 2.3.0**：官方明确支持 Flink 1.13/1.14/1.15/1.16.0，且要求 Java 8+。CDC 2.4+ 要求 Flink 1.17+（需要 JDK 11）。

**Iceberg 1.3.1**：提供 `iceberg-flink-runtime-1.15` 和 `iceberg-spark-runtime-3.3`，Flink 端和 Spark 端使用同一版本。Iceberg 1.4+ 主要测试目标为 JDK 11+。

**Spark 3.3.1**：Spark 3.3 线，完全兼容 JDK 8，纯净版（without-hadoop）通过 `SPARK_DIST_CLASSPATH` 注入系统 Hadoop 类路径。

**Kafka 3.6.2**：Kafka 3.x 全系仍支持 JDK 8（自 3.0 起标记为 deprecated），Kafka 4.0 才正式移除 JDK 8 支持。3.6.2 是该线稳定版本，Scala 2.12 编译。

### 第一阶段：基础设施（2-3 天）—— ✅ 已完成

```

     - 三节点集群
```

### 8.2 SeaTunnel 配置

> SeaTunnel 将 Kafka 消息以 JSON 格式写入 HDFS，与 ODS 表 `ods_log_inc` 的 `JsonSerDe` 直接匹配。STRUCT/ARRAY 类型字段由 Hive 在读取时自动解析。

```hocon
# config/jobs/kafka_log_to_hdfs.conf
env {
  job.mode = "STREAMING"
  parallelism = 1
  checkpoint.interval = 60000
}

source {
  Kafka {
    topic = "topic_log"                                          # 2.3.13 用 topic（单数）
    bootstrap.servers = "hadoop100:9092,hadoop101:9092,hadoop102:9092"
    consumer.group = "seatunnel_log_consumer"
    format = json
    start_mode = "group_offsets"
    plugin_output = "kafka_source"                               # 2.3.13 新参数名
  }
}

sink {
  HdfsFile {
    plugin_input = "kafka_source"                                # 2.3.13 新参数名
    fs.defaultFS = "hdfs://hadoop100:8020"                       # 2.3.13 必须显式指定
    path = "/warehouse/gmall/ods/ods_log_inc"
    have_partition = true
    partition_dir_expression = "dt=${now}"
    file_format = "json"
    filename_time_format = "HHmmss"
    max_rows_in_memory = 1000000
    is_enable_transaction = true
  }
}
```

### 8.3 Flink Job 1: MySQL CDC → Kafka

完整脚本见 `cdc_to_kafka.sql`（304行），结构：

- **kafka_sink**：upsert-kafka 通用 sink，`(table_name, row_id, data STRING)`，topic=topic_db
- **13 个 mysql-cdc Source**：cdc_order_info ~ cdc_cart_info，scan.startup.mode=initial
- **13 条 INSERT**：`JSON_OBJECT()` 将每列序列化写入 data 字段

> **设计要点**：Flink CDC 2.3.0 SQL 模式下 CDC Source 暴露的是 MySQL 实际列，不暴露 debezium 元数据 (op/before/after)，因此用 upsert-kafka + JSON_OBJECT() 而非 debezium-json。

### 8.4 Flink Job 2: Kafka → Iceberg Upsert

完整脚本见 `kafka_to_iceberg.sql`（72行），结构：

- **kafka_source**：消费 topic_db，`(table_name, row_id, data STRING)`，earliest-offset
- **13 个 Iceberg Sink 表**：HadoopCatalog (catalog-type='hadoop')，write.upsert.enabled=true
- **13 条 INSERT**：`JSON_VALUE(data, '$.field')` 解析 + CAST，WHERE table_name 过滤，dt 分区键从 create_time 提取

> **设计要点**：用 HadoopCatalog 而非 HiveCatalog，避免 Flink 依赖 Hive Metastore client JAR。批次标记 `@@BATCH@@` 和 `@@SRC_DEF_END@@` 供 run_job2.sh 的 csplit 拆分。

## 十、注意事项与避坑点

### 10.1 JDK 版本相关

**坑 1：不要用 JDK 11 以上的组件版本。**
Flink 1.16+、Spark 3.4+、Flink CDC 3.0+ 均不兼容 JDK 8，强行运行会报 `UnsupportedClassVersionError`。Kafka 3.x 兼容 JDK 8，但 Kafka 4.0 将移除。严格按照第五章的版本安装。

**坑 2：编译自己的代码也要注意 target 版本。**
如果自己写 Java/Scala 代码（如自定义 Flink Job），Maven/Gradle 中务必设置 `target=1.8`。

### 10.2 资源相关

**坑 3：4GB 节点 JVM 堆控制。**
每个 Java 进程默认堆可能 256MB~1GB。多个进程叠加极易 OOM。务必显式设置：
```bash
# Hadoop
export HADOOP_HEAPSIZE=256
# Kafka
export KAFKA_HEAP_OPTS="-Xmx512M -Xms256M"
# Flink TaskManager
taskmanager.memory.process.size: 1024m
# ZooKeeper
export JVMFLAGS="-Xmx256M"
```

**坑 4：SeaTunnel 和 Flink 不要同时抢占 YARN 资源。**
SeaTunnel 建议用 local 模式或独立部署，不提交到 YARN，避免和 Flink 竞争。Flink Job 提交到 YARN。

### 10.3 Iceberg 相关

**坑 5：Iceberg 表必须用 format-version=2。**
v2 才支持行级删除（DELETE 操作对应的 Upsert 需要）。建表时务必加 `'format-version' = '2'`。

**坑 6：Hive Metastore 中 Iceberg 表的 Catalog 配置。**
Spark SQL 和 Flink SQL 中都需要配置 Iceberg catalog，指向同一个 Hive Metastore：
```bash
# Spark SQL 启动参数
--conf spark.sql.extensions=org.apache.iceberg.spark.extensions.IcebergSparkSessionExtensions
--conf spark.sql.catalog.hive_prod=org.apache.iceberg.spark.SparkCatalog
--conf spark.sql.catalog.hive_prod.type=hive
```

**坑 7：Flink Iceberg Upsert 需要主键。**
Flink 的 `write.upsert.enabled=true` 依赖表定义了 `PRIMARY KEY`。建表 DDL 和 Flink SQL Sink 定义中都要声明主键。

**坑 8：Compaction 不能忘。**
Flink 持续小批写入 Iceberg 会积累大量小文件。如果一周不跑 Compaction，查询性能会严重下降。建议每日凌晨在 DolphinScheduler 最后一步执行。

### 10.4 Flink CDC 相关

**坑 9：MySQL binlog 格式必须是 ROW。**
```sql
-- MySQL 中确认
SHOW VARIABLES LIKE 'binlog_format';  -- 必须是 ROW
```
Statement 或 Mixed 格式下 Flink CDC 无法正确解析数据变更。

**坑 10：首日全量快照时 MySQL 压力。**
`scan.startup.mode = 'initial'` 会对每张表执行 `SELECT *`，13 张表同时全量快照可能打满 MySQL 连接池。建议：
- 分批次执行（先 5 张表全量，完成后下 5 张）
- 或者单独用 DataX 做首日全量（速度快），然后 Flink CDC 用 `specific-offset` 从特定位点开始追增量

**坑 11：Flink CDC 的表名匹配。**
如果配置 `'table-name' = '.*'` 会全库采集，包括全量表。必须按表名精确指定 13 张增量表，避免全量表的数据也走一遍 CDC。

### 10.5 数据相关

**坑 12：日志 JSON 中的逗号缺失。**
模拟数据生成脚本务必检查：`common.sid` 和 `common.uid` 之间、`displays` 数组中 `pos_id` 后面是否缺逗号。Filebeat 的 `json.add_error_key: true` 会标记解析失败的条目，可在 Kafka 消费端过滤出来排查。

**坑 13：cart_info 只走增量。**
cart_info 已从全量同步列表中移除。DataX 全量脚本中不要包含 cart_info，否则会和 Flink CDC 写入的 Iceberg 表数据不一致。

**坑 14：增量表首日全量不要漏。**
如果 13 张增量表中有某张没有完成首日全量初始化，则日常增量追数时 Iceberg 表中缺失历史数据，DWD 层的 JOIN 会丢失关联。

### 10.6 组件兼容性

**坑 15：Flink 和 Spark 的 Iceberg JAR 不能混用。**
Flink 用 `iceberg-flink-runtime-1.15-1.3.1.jar`，Spark 用 `iceberg-spark-runtime-3.3_2.12-1.3.1.jar`。两个 JAR 不同，但版本号一致（都是 1.3.1），格式兼容。

**坑 16：Hive Metastore 版本。**
如果使用 Hive 2.x 的 Metastore，Iceberg 1.3.x 可能不完全兼容。建议使用 Hive 3.x 的 Metastore（可独立部署，不需要完整 Hive）。

### 10.7 实施顺序

**坑 17：不要跳过验证环节。**
每个组件安装后必须验证。常见跳过的验证导致的问题：
- ZooKeeper 未验证（Kafka 起不来，但不报明显错误）
- MySQL binlog 未验证（Flink CDC 看似正常但无数据）
- Iceberg 表未验证（Flink Job 2 运行但不报错也不写数据——因为表不存在或格式不对）
