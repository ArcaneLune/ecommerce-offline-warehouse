# 电商离线数仓 —— ADS 报表数据导出实操文档

> ADS → MySQL 导出，使用 DataX HDFSReader + MySQLWriter，配置由自建 datax-config-generator JAR 自动生成。最后更新：2026-06-26。

---

## 一、导出架构

```
Hive ADS 表（TextFile, TAB 分隔, 不分区）
    │
    │  DataX HDFSReader 读取
    ▼
MySQL gmall_report 库（16 张报表表）
```

| 项目 | 值 |
|------|-----|
| Hive ADS 格式 | TextFile，TAB 分隔，路径 `/warehouse/gmall/ads/{表名}/` |
| MySQL 目标库 | `gmall_report`（hadoop101:3306，用户 root / root） |
| 导出工具 | DataX v202309 |
| Reader | hdfsreader（读取 HDFS 目录下全部数据文件） |
| Writer | mysqlwriter |
| writeMode | replace（全量覆盖，保证与 Hive 数据一致） |
| 配置生成 | 自建 `datax-config-generator-1.0-SNAPSHOT-jar-with-dependencies.jar` |


## 二、操作步骤

### 2.1 在 MySQL 中建库建表

```bash
# 上传 ads_mysql_ddl.sql 到 hadoop101
mysql -h hadoop101 -u root -proot < /opt/software/ads_mysql_ddl.sql
```
> 参考sql/ads/ads_mysql_ddl.sql

验证：

```bash
mysql -h hadoop101 -u root -proot -e "USE gmall_report; SHOW TABLES;"
# 期望输出 16 张表
```

### 2.2 修改配置生成器参数

编辑 `/opt/software/datax-generator/configuration.properties`：

```properties
mysql.username=root
mysql.password=root
mysql.host=hadoop101
mysql.port=3306
mysql.database.import=gmall
# 从 HDFS 导出进入的 MySQL 数据库
mysql.database.export=gmall_report
# 全量导入的表（DataX MySQL→Hive ODS）
mysql.tables.import=activity_info,activity_rule,base_trademark,cart_info,base_category1,base_category2,base_category3,coupon_info,sku_attr_value,sku_sale_attr_value,base_dic,sku_info,base_province,spu_info,base_region,promotion_pos,promotion_refer
# 导出表：空串 = 取 gmall_report 库全部 16 张表
mysql.tables.export=
is.seperated.tables=0
hdfs.uri=hdfs://hadoop100:8020
import_out_dir=/opt/module/datax/job/import
# DataX 导出配置文件存放路径
export_out_dir=/opt/module/datax/job/export
```

> 关键字段说明：
> - `mysql.tables.export=` 留空 → JAR 自动扫描 `gmall_report` 库的全部表
> - `hdfs.uri=hdfs://hadoop100:8020` → NameNode 在 hadoop100
> - `mysql.host=hadoop101` → MySQL 在 hadoop101

### 2.3 生成 DataX 导出配置文件

```bash
cd /opt/software/datax-generator/
java -jar datax-config-generator-1.0-SNAPSHOT-jar-with-dependencies.jar

# 确认生成 16 个 JSON
ls /opt/module/datax/job/export/
```

JAR 会：
1. 连接 MySQL `gmall_report` 库，读取 16 张表的元数据（列名、类型）
2. 为每张表生成一个 DataX JSON 配置文件
3. JSON 中 HDFS path 为 `/warehouse/gmall/ads/{表名}/`，writeMode 为 replace

### 2.4 执行导出

```bash
# 串行执行 16 张表导出
# ADS 表不分区，不需要传参数，JSON 中路径已写死
for job in /opt/module/datax/job/export/gmall_report.*.json; do
    tbl=$(basename $job .json | sed 's/gmall_report\.//')
    echo ">>> 导出: ${tbl}"
    python /opt/module/datax/bin/datax.py \
        -p"-Dexportdir=/warehouse/gmall/ads/${tbl}" \
        $job
    if [ $? -ne 0 ]; then
        echo "!!! ${tbl} 导出失败"
    fi
done

# 或先测试单张表
python /opt/module/datax/bin/datax.py \
    -p"-Dexportdir=/warehouse/gmall/ads/ads_user_stats" \
    /opt/module/datax/job/export/gmall_report.ads_user_stats.json
```

> 注意：JSON 中 path 字段是 `${exportdir}` 占位符，运行时通过 `-p"-Dexportdir=..."` 传入实际路径。ADS 表不分区，路径直接取表目录即可，无需 `dt=` 子目录。

## 三、文件清单

| 文件 | 用途 |
|------|------|
| `configuration.properties` | DataX 配置生成器参数（已含导入/导出全部配置） |
| `datax-config-generator-1.0-SNAPSHOT-jar-with-dependencies.jar` | 自建 JAR，自动扫描 MySQL 表生成 DataX JSON |
| `ads_mysql_ddl.sql` | MySQL gmall_report 库 16 张表建表 |
| `/opt/module/datax/job/export/gmall_report.*.json` | JAR 生成的 16 个 DataX 导出配置文件 |

## 四、DataX JSON 模板说明

以 `ads_traffic_stats_by_channel` 为例，JAR 生成的 JSON 结构：

```json
{
    "job": {
        "content": [{
            "reader": {
                "name": "hdfsreader",
                "parameter": {
                    "column": ["*"],
                    "defaultFS": "hdfs://hadoop100:8020",
                    "encoding": "UTF-8",
                    "fieldDelimiter": "\t",
                    "fileType": "text",
                    "nullFormat": "\\N",
                    "path": "${exportdir}"
                }
            },
            "writer": {
                "name": "mysqlwriter",
                "parameter": {
                    "column": ["dt","recent_days","channel","uv_count","avg_duration_sec","avg_page_count","sv_count","bounce_rate"],
                    "connection": [{
                        "jdbcUrl": "jdbc:mysql://hadoop101:3306/gmall_report?useUnicode=true&characterEncoding=utf-8",
                        "table": ["ads_traffic_stats_by_channel"]
                    }],
                    "password": "000000",
                    "username": "root",
                    "writeMode": "replace"
                }
            }
        }],
        "setting": {
            "errorLimit": { "percentage": 0.02, "record": 0 },
            "speed": { "channel": 3 }
        }
    }
}
```

> - `path: "${exportdir}"` 是占位符，运行时通过 `-p"-Dexportdir=..."` 替换
> - `writeMode: "replace"` → 先 TRUNCATE 目标表再写入，保证数据与 Hive 一致
> - `column` 列表由 JAR 自动从 MySQL 元数据获取，无需手动填写

## 五、每日调度集成

DolphinScheduler 中 ads_export 任务节点的命令：

```bash
#!/bin/bash
dt=$(date +%Y-%m-%d)

# 1. 生成最新 DataX 配置（确保列信息与 MySQL 同步）
cd /opt/module/gen_datax_config
java -jar datax-config-generator-1.0-SNAPSHOT-jar-with-dependencies.jar

# 2. 串行执行 16 张表导出
for job in /opt/module/datax/job/export/gmall_report.*.json; do
    tbl=$(basename $job .json | sed 's/gmall_report\.//')
    python /opt/module/datax/bin/datax.py \
        -p"-Dexportdir=/warehouse/gmall/ads/${tbl}" \
        $job
    [ $? -ne 0 ] && echo "!!! ${tbl} 导出失败"
done
```

## 六、与 ODS 全量导入的关键区别

| 项目 | ODS 全量导入（MySQL→Hive） | ADS 导出（Hive→MySQL） |
|------|:---:|:---:|
| 方向 | MySQL → HDFS | HDFS → MySQL |
| DataX Reader | mysqlreader | **hdfsreader** |
| DataX Writer | hdfswriter | **mysqlwriter** |
| 建分区目录 | ✅ `hdfs dfs -mkdir -p dt=...` | ❌ ADS 不分区 |
| 注册分区 | ✅ `MSCK REPAIR TABLE` | ❌ 不分区不需注册 |
| 传参 | `-p"-Dtargetdir=.../dt=${dt}"` | `-p"-Dexportdir=..."`，不含 dt |
| 配置文件目录 | `/opt/module/datax/job/import/` | `/opt/module/datax/job/export/` |
