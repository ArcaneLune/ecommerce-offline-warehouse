# 电商离线数仓 —— ADS 报表数据导出实操文档

> ADS → MySQL 导出，使用 DataX HDFSReader + MySQLWriter。最后更新：2026-06-26。

---

## 一、导出架构

```
Hive ADS 表（TextFile, TAB 分隔）
    │
    │  DataX HDFSReader 读取
    ▼
MySQL gmall_report 库（16 张报表表）
```

| 项目 | 值 |
|------|-----|
| Hive ADS 格式 | TextFile，TAB 分隔，路径 `/warehouse/gmall/ads/{表名}/` |
| MySQL 目标库 | `gmall_report`（hadoop101:3306） |
| 导出工具 | DataX |
| Reader | hdfsreader |
| Writer | mysqlwriter |
| writeMode | replace（全量覆盖当日数据） |

## 二、与旧项目差异

| 旧项目 | 新架构 |
|--------|--------|
| MySQL 在 hadoop102 | MySQL 在 **hadoop101** |
| `ads_user_change` 计数字段为 `varchar(16)` | 修正为 **`bigint(20)`** |
| 导出路径硬编码在 JSON | 脚本自动生成，路径 `/warehouse/gmall/ads/{表名}/` |

## 三、操作步骤

### 3.1 在 MySQL 中建库建表

```bash
# 上传 ads_mysql_ddl.sql 到 hadoop101
mysql -h hadoop101 -u root -p000000 < /opt/software/ads_mysql_ddl.sql
```

验证：
```bash
mysql -h hadoop101 -u root -p000000 -e "USE gmall_report; SHOW TABLES;"
# 期望输出 16 张表
```

### 3.2 执行 DataX 导出

```bash
# 上传 ads_export.sh 到 hadoop101，赋予执行权限
chmod +x /opt/software/ads_export.sh

# 执行导出（默认当天日期）
bash /opt/software/ads_export.sh

# 或指定日期
bash /opt/software/ads_export.sh 2026-06-26
```

脚本会：
1. 在 `/opt/module/datax/job/export/` 下为每张表生成 JSON 配置文件
2. 逐个执行 `python datax.py`
3. 遇到失败的跳过，继续下一张

### 3.3 验证 MySQL 数据

```bash
mysql -h hadoop101 -u root -p000000 -e "
USE gmall_report;
SELECT 'ads_traffic_stats_by_channel', count(*) FROM ads_traffic_stats_by_channel
UNION ALL SELECT 'ads_user_change', count(*) FROM ads_user_change
UNION ALL SELECT 'ads_user_retention', count(*) FROM ads_user_retention
UNION ALL SELECT 'ads_user_stats', count(*) FROM ads_user_stats
UNION ALL SELECT 'ads_user_action', count(*) FROM ads_user_action
UNION ALL SELECT 'ads_order_by_province', count(*) FROM ads_order_by_province
UNION ALL SELECT 'ads_coupon_stats', count(*) FROM ads_coupon_stats;
"
```

## 四、文件清单

| 文件 | 用途 |
|------|------|
| `ads_mysql_ddl.sql` | MySQL gmall_report 库 16 张表建表 |
| `ads_export.sh` | DataX 配置生成 + 导出执行脚本 |

## 五、DataX JSON 模板说明

脚本为每张表生成如下 JSON（以 ads_traffic_stats_by_channel 为例）：

```json
{
    "job": {
        "content": [{
            "reader": {
                "name": "hdfsreader",
                "parameter": {
                    "column": ["*"],
                    "defaultFS": "hdfs://hadoop100:8020",
                    "encoding": "UTF-8",
                    "fieldDelimiter": "\t",
                    "fileType": "text",
                    "nullFormat": "\\N",
                    "path": "/warehouse/gmall/ads/ads_traffic_stats_by_channel"
                }
            },
            "writer": {
                "name": "mysqlwriter",
                "parameter": {
                    "column": ["dt","recent_days","channel",...],
                    "connection": [{
                        "jdbcUrl": "jdbc:mysql://hadoop101:3306/gmall_report?...",
                        "table": ["ads_traffic_stats_by_channel"]
                    }],
                    "password": "000000",
                    "username": "root",
                    "writeMode": "replace"
                }
            }
        }],
        "setting": {
            "errorLimit": { "percentage": 0.02, "record": 0 },
            "speed": { "channel": 3 }
        }
    }
}
```

> `writeMode: "replace"` — 每次导出前清空目标表再写入，保证数据与 Hive 一致。
