## 一、全量同步链路总览

### 1.1 数据流向

```
MySQL 8.0.39              DataX v202309              HDFS                       Hive/Spark
(hadoop101)               (hadoop101)                (HDFS集群)                 (hadoop100)
    ↓                         ↓                         ↓                           ↓
29 张业务表                 mysqlreader 读取             /warehouse/ods/            Parquet 格式
(全量表 16 张)      →       hdfswriter 写入     →      {表名}_full/        →      Hive 原生表
                           每日凌晨调度                 dt=YYYY-MM-DD/              ods_xxx_full
                                                       Parquet 文件                16 张
```

### 1.2 关键组件

| 组件 | 版本 | 节点 | 角色 |
|------|------|:---:|------|
| MySQL | 8.0.39 | hadoop101 | 业务数据源，存储 29 张业务表 |
| DataX | v202309 | hadoop101 | 批同步工具，mysqlreader → hdfswriter |
| HDFS | 3.3.6 | 三节点 | 文件存储 |
| Hive Metastore | 3.1.3 | hadoop100 | 元数据管理 |
| DolphinScheduler | 3.1.9 | hadoop102 | 每日凌晨调度 |

### 1.4 与日志链路的区别

| 维度 | 日志链路 | 全量同步链路 |
|------|---------|------------|
| 数据源 | 模拟 .log 文件 | MySQL 业务表 |
| 采集方式 | Filebeat → Kafka → SeaTunnel（持续流） | DataX（每日定时批） |
| 存储格式 | JSON（JsonSerDe） | Parquet |
| 同步策略 | 增量追加 | 每日全量覆盖 |
| 运行模式 | 常驻进程 | DolphinScheduler 每日调度一次 |
| 数据量 | 16 张表 | 日志每天约 10-30MB |
| 调度依赖 | 无（常驻） | 依赖 DolphinScheduler 工作流 |

## 二、第一步：MySQL 业务数据导入

**执行节点**：hadoop101
**目标**：在 MySQL 中创建 gmall 数据库全部 29 张表，导入模拟业务数据。

### 操作步骤

##### 1. 创建数据库并执行建表语句

```bash
# hadoop101 上执行
mysql -uroot -p -e "CREATE DATABASE IF NOT EXISTS gmall DEFAULT CHARACTER SET utf8mb4 DEFAULT COLLATE utf8mb4_general_ci"
mysql -uroot -p gmall < /opt/software/gmall_schema.sql
```

##### 2. 运行数据模拟生成器

```bash
# 初始化模式：建立维度数据 + 初始交易
python3 /home/hadoop/bin/mysql_data_generator.py --mode init --host hadoop101 --password root

# 持续模式：后台不断生成新交易
nohup python3 /home/hadoop/bin/mysql_data_generator.py --mode continuous --host hadoop101 --batch 50 --interval 5 --password root > /home/hadoop/mysql_gen.log 2>&1 &
```

### 验证

```bash
mysql -uroot -p gmall -e "SELECT COUNT(*) FROM activity_info"   # 20
mysql -uroot -p gmall -e "SELECT COUNT(*) FROM sku_info"         # ~500
mysql -uroot -p gmall -e "SELECT COUNT(*) FROM user_info"        # 2000
mysql -uroot -p gmall -e "SELECT COUNT(*) FROM order_info"       # > 0
```

## 三、第二步：ODS 全量表建表（16 张）

**执行节点**：hadoop100
**工具**：`hive` CLI

### 操作步骤

##### 1. 创建 HDFS 目录

```bash
for t in activity_info activity_rule base_category1 base_category2 base_category3 \
         base_dic base_province base_region base_trademark coupon_info \
         sku_attr_value sku_info sku_sale_attr_value spu_info \
         promotion_pos promotion_refer; do
    hdfs dfs -mkdir -p /warehouse/gmall/ods/ods_${t}_full
done
```

##### 2. 执行建表 DDL

```bash
# 上传原始 DDL 文件（带 TBLPROPERTIES 版本）到 /opt/software/
# 用 hive CLI 执行
hive -f /opt/software/ods_full_tables_ddl.sql
```
> 参考sql/ods/ods_full_tables_ddl.sql

##### 3. 启动 HiveServer2

```bash
nohup hive --service metastore >> /opt/module/hive-3.1.3/logs/metastore.log 2>&1 &
nohup hive --service hiveserver2 >> /opt/module/hive-3.1.3/logs/hiveserver2.log 2>&1 &
sleep 3
netstat -tlnp | grep 10000
```

### 验证

```bash

# 应列出 16 张表

beeline -u jdbc:hive2://hadoop100:10000 -n hadoop -e "DESC ods_activity_info_full"
```

> **工具分工**：DDL 用 `hive`，查询用 `beeline`，Spark 负责底层计算。

## 四、第三步：DataX 批量生成配置 + 全量同步

**执行节点**：hadoop101
**目标**：批量生成 16 张全量表的 DataX JSON 配置文件，全部跑通。

### DataX JSON 配置生成

##### 1. 上传 JAR 包

将 `datax-config-generator-1.0-SNAPSHOT-jar-with-dependencies.jar` 上传到 hadoop101 的 `/opt/software/`。
参考datax-config-generator/target/datax-config-generator-1.0-SNAPSHOT-jar-with-dependencies.jar

##### 2. 创建配置文件

```bash
mkdir -p /opt/software/datax-generator
cd /opt/software/datax-generator

cat > configuration.properties << 'EOF'
mysql.username=root
mysql.password=root
mysql.host=hadoop101
mysql.port=3306
mysql.database.import=gmall
mysql.tables.import=activity_info,activity_rule,base_trademark,base_category1,base_category2,base_category3,coupon_info,sku_attr_value,sku_sale_attr_value,base_dic,sku_info,base_province,spu_info,base_region,promotion_pos,promotion_refer
is.seperated.tables=0
hdfs.uri=hdfs://hadoop100:8020
import_out_dir=/opt/module/datax/job/import
EOF
```

> **注意**：`mysql.tables.import` 中不包含 `cart_info`——该表只走 Flink CDC 增量同步。

##### 3. 执行生成

```bash
# 将 JAR 移动到同目录
mv /opt/software/datax-config-generator-1.0-SNAPSHOT-jar-with-dependencies.jar /opt/software/datax-generator/

cd /opt/software/datax-generator/
java -jar datax-config-generator-1.0-SNAPSHOT-jar-with-dependencies.jar
```

生成后，16 个 JSON 配置文件位于 `/opt/module/datax/job/import/`。


### 运行 DataX 同步

```bash
# 1. 创建当天分区目录（DataX 不会自动建目录，必须先建）
dt=$(date +%Y-%m-%d)
for t in activity_info activity_rule base_trademark base_category1 base_category2 \
         base_category3 coupon_info sku_attr_value sku_sale_attr_value base_dic \
         sku_info base_province spu_info base_region promotion_pos promotion_refer; do
    hdfs dfs -mkdir -p /warehouse/gmall/ods/ods_${t}_full/dt=${dt}
done

# 2. 串行执行 16 张表同步
for job in /opt/module/datax/job/import/gmall.*.json; do
    tbl=$(basename $job .json | sed 's/gmall\.//')
    echo ">>> ${tbl}"
    python /opt/module/datax/bin/datax.py -p"-Dtargetdir=/warehouse/gmall/ods/ods_${tbl}_full/dt=${dt}" $job
done

# 3. 批量注册分区（★ DataX 不会自动注册，必须执行）
dt=$(date +%Y-%m-%d)
for t in activity_info activity_rule base_trademark base_category1 base_category2 \
         base_category3 coupon_info sku_attr_value sku_sale_attr_value base_dic \
         sku_info base_province spu_info base_region promotion_pos promotion_refer; do
    beeline -u jdbc:hive2://hadoop100:10000 -n hadoop --silent=true -e "MSCK REPAIR TABLE ods_${t}_full"
    echo "registered: ods_${t}_full"
done
```

> **注意**：DataX hdfswriter 不会自动创建分区目录，必须先 `hdfs dfs -mkdir`。数据写入后也不会自动注册分区，必须 `MSCK REPAIR TABLE`，否则查询返回 `SELECT count(*) = 0`。

### 验证

```bash
# hadoop100 上查询
beeline -u jdbc:hive2://hadoop100:10000 -n hadoop -e "SELECT count(*) FROM ods_activity_info_full"
beeline -u jdbc:hive2://hadoop100:10000 -n hadoop -e "SELECT count(*) FROM ods_sku_info_full"
```

# ===== hadoop101：MySQL + 全量同步 =====
sudo systemctl start mysqld
dt=$(date +%Y-%m-%d)
for t in activity_info activity_rule base_trademark base_category1 base_category2 \
         base_category3 coupon_info sku_attr_value sku_sale_attr_value base_dic \

done

# ===== hadoop100：注册分区=====
for t in activity_info activity_rule base_trademark base_category1 base_category2 \
